
Визуализация пайплайна
1. Метод визуализации

Для визуализации пайплайна был построен DAG в формате DOT и затем преобразован в SVG с помощью Graphviz.

Файлы:

results/reflow/mapping_qc_dag.dot;
results/reflow/mapping_qc_dag.svg.

Команда генерации SVG:

dot -Tsvg results/reflow/mapping_qc_dag.dot > results/reflow/mapping_qc_dag.svg
2. Что показывает DAG

DAG показывает зависимости между вычислительными шагами:

FASTQ-файлы нужны для FastQC и BWA MEM;
reference genome нужен для индексации и выравнивания;
результат BWA MEM нужен для samtools view;
BAM нужен для samtools flagstat;
результат flagstat нужен для принятия решения по mapped reads;
при статусе OK запускаются sort, index и freebayes;
итогом является VCF.
3. Отличие DAG от блок-схемы

Блок-схема обычно описывает алгоритм как последовательность инструкций и условий. Она удобна для объяснения логики человеку.

DAG описывает вычислительные зависимости между задачами. Он показывает, какие задачи могут быть выполнены только после появления определённых входных данных. Такой формат ближе к workflow systems, потому что workflow framework планирует запуск задач не по строкам кода, а по зависимостям между входами и выходами.

В данной работе DAG используется для представления структуры пайплайна как набора связанных вычислительных шагов.
